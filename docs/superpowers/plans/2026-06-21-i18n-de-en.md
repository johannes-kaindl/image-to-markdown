# UI-Lokalisierung EN/DE (i18n) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Alle nutzersichtbaren Plugin-Strings sprechen EN/DE nach Obsidian-Sprach-Einstellung; das Muster wird als Workspace-Standard in `_docs` verankert.

**Architecture:** Pures `src/i18n.ts`-Modul (kein obsidian/DOM) mit `STRINGS{en,de}`, `t()`-Lookup (Fallback `currentLang→en→key`, `{0}`-Interpolation) und modul-lokaler `currentLang`. Detektion in `main.ts` (`setLang(pickLang(getLanguage()))` beim `onload`). EN kanonisch.

**Tech Stack:** TypeScript (strict), esbuild, vitest + happy-dom, Obsidian Plugin API.

## Global Constraints

- **EN kanonisch**, dreistufiger Fallback `currentLang → en → key`.
- **Reiner Kern** (`img_to_md.ts`, `capabilities.ts`, `img_to_md_state.ts`, `i18n.ts`) ohne `obsidian`/DOM-Import — Node-testbar (PROF-OBS-03/04).

```ts
export type Lang = "en" | "de";
let currentLang: Lang = "en";
export function pickLang(raw?: string | null): Lang {
  return raw && raw.toLowerCase().startsWith("de") ? "de" : "en";
}
export function setLang(lang: Lang): void { currentLang = lang; }
export function getLang(): Lang { return currentLang; }
type Dict = Record<string, string>;
const EN: Dict = { /* alle Keys, EN-Werte */ };
const DE: Dict = { /* alle Keys, DE-Werte */ };
const STRINGS: Record<Lang, Dict> = { en: EN, de: DE };
export function t(key: string, ...args: (string | number)[]): string {
  const raw = STRINGS[currentLang][key] ?? STRINGS.en[key] ?? key;
  return raw.replace(/\{(\d+)\}/g, (_m, i) => {
    const v = args[Number(i)];
    return v === undefined ? `{${i}}` : String(v);
  });
}
export function defaultVisionPrompt(): string { return t("prompt.default"); }
```
